Getting Started

The Ailevate AMI Agentic Agent Creator is a no-code platform that enables users to create specialized AI agents without programming experience. This documentation will guide you through the process of creating, configuring, and deploying your own custom agents.

What Are Ailevate AMI Agentic Agents?

Ailevate AMI (Adaptive Machine Intelligence) Agentic Agents are specialized AI assistants designed to perform specific tasks or serve particular domains. These agents are powered by large language models (LLMs) but are configured with specific capabilities, knowledge bases, and behaviors to make them experts in their designated areas.

Key Concepts

  • Agent: An AI assistant with a specific purpose and set of capabilities
  • Capabilities: The specific skills and functions an agent can perform
  • Knowledge Base: Domain-specific information the agent has access to
  • Model Settings: Configuration of the underlying AI model
  • YAML Configuration: The structured format that defines an agent's properties

Agent Creation Process

Creating a new agent involves several steps:

Step 1: Choose a Template

Start with a pre-configured template or build from scratch. Templates provide a foundation with pre-selected capabilities and settings for specific use cases.

Step 2: Define Basic Information

Set your agent's name, description, and category. This information helps users understand what your agent does and how it can assist them.

Step 3: Configure Capabilities

Select the specialized skills and knowledge bases your agent will have. These determine what tasks your agent can perform and what information it can access.

Step 4: Adjust Model Settings

Configure the underlying AI model parameters such as model type, context size, and temperature. These settings affect how your agent processes information and generates responses.

Step 5: Preview and Export

Test your agent's behavior through simulation and export the YAML configuration. This configuration can be imported into Dify.ai for deployment.

Model Settings

Model settings determine how your agent processes information and generates responses. Configuring these properly is crucial for optimizing your agent's performance for specific domains.

Model Type

The type of AI model that powers your agent:

  • LLM (Large Language Model): For text generation and conversation
    • Best for: Customer support, content creation, medical coding review
    • Example: A medical coding agent that can understand complex health documentation and provide accurate coding recommendations
  • Text Embedding: For semantic search and retrieval
    • Best for: Knowledge retrieval, document search, similar case finding
    • Example: A legal research agent that can find similar precedent cases based on semantic similarity
  • Rerank: For reordering search results by relevance
    • Best for: Result prioritization, content filtering, relevance sorting
    • Example: A research agent that improves search results by prioritizing the most relevant documents
Pro Tip: For complex workflows, consider creating multiple specialized agents with different model types that work together. For example, a Text Embedding agent to find relevant documents paired with an LLM agent to generate responses based on those documents.

YAML Structure

The YAML configuration defines all aspects of your agent. Here's an example of the structure:

model: agent-identifier
label:
  en_US: Agent Display Name
model_type: llm
features:
- agent-thought
- tool-use
description:
  en_US: Detailed description of what the agent does
knowledge_bases:
- knowledge-base-1
- knowledge-base-2
model_properties:
  mode: chat
  context_size: 16000
parameter_rules:
- name: temperature
  use_template: temperature
  default: 0.7
- name: top_p
  use_template: top_p
  default: 0.9
- name: max_tokens
  use_template: max_tokens
  default: 1024
capabilities:
  capability_1:
    enabled: true
    description: Description of the capability
  capability_2:
    enabled: true
    description: Description of the capability
tools:
  tool_1:
    enabled: true
    description: Description of the tool
  tool_2:
    enabled: true
    description: Description of the tool
pricing:
  input: '0.00'
  output: '0.00'
  unit: '0.000001'
  currency: USD

Key Sections

  • model: Unique identifier for your agent
  • label: Display name in different languages
  • model_type: Type of AI model (llm, text-embedding, rerank)
  • features: List of enabled advanced features
  • description: Detailed description of what the agent does
  • knowledge_bases: List of knowledge sources the agent can access
  • model_properties: Core model configuration
  • parameter_rules: Model invocation parameters
  • capabilities: Specialized skills the agent has
  • tools: External tools the agent can use
  • pricing: Cost information for usage

Dify.ai Integration

The Ailevate AMI Agentic Agent Creator generates YAML configurations that are compatible with Dify.ai, allowing you to deploy your custom agents to a production environment.

Integration Steps

  1. Export the YAML configuration from the AMI Agentic Agent Creator
  2. Log in to your Dify.ai account
  3. Navigate to "Models" section
  4. Select "Add Custom Model"
  5. Upload the YAML configuration file
  6. Configure any additional settings
  7. Deploy your agent
Note: Make sure your Dify.ai environment has access to the required AI models and knowledge bases specified in your configuration.

Adaptive OODA Framework

The Ailevate AMI Agentic Agent Creator implements the OODA (Observe, Orient, Decide, Act) loop methodology to create truly adaptive AI agents that learn and improve over time.

The OODA Loop Cycle

A continuous feedback mechanism for adaptive AI

OBSERVE

Collect data from interactions and outcomes

ORIENT

Analyze data against defined goals

DECIDE

Select optimal response strategies

ACT

Execute and measure effectiveness

Configuring OODA Parameters

In Step 5 of the Agent Creator, you can fine-tune how your agent adapts:

Business Goals

Define what success looks like for your agent:

  • Primary Goal: The main objective (e.g., "Reduce coding errors by 30%")
  • Secondary Goals: Additional objectives to balance (e.g., "Maintain throughput")
  • Constraints: Limitations the agent must respect (e.g., "HIPAA compliance")
Adaptation Parameters

Control how quickly and deeply your agent adapts:

  • Adaptation Level: Parameter adjustment, strategy selection, or complex learning
  • Adaptation Rate: Conservative (stable) to aggressive (rapid learning)
  • Feedback Interval: How frequently the agent updates its behavior

The OODA Loop works through continuous cycles of adaptation:

In Practice: When a medical coding agent processes an insurance claim:

  1. Observe: The agent collects data about the coding task (user inputs, previous code assignments, rejection feedback).
  2. Orient: It analyzes this data against accuracy goals, identifying patterns in successful vs. rejected codes.
  3. Decide: The agent selects the optimal coding strategy based on specialty, complexity, and previous successes.
  4. Act: It implements the strategy by recommending specific codes and explaining rationale.

After receiving feedback (acceptance or rejection of codes), the cycle begins again with new data, gradually improving performance. This occurs automatically without human intervention or model retraining.

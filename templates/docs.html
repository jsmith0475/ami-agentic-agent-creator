Getting Started

The Ailevate AMI Agent Creator is a no-code platform that enables users to create specialized AI agents without programming experience. This documentation will guide you through the process of creating, configuring, and deploying your own custom agents.

What Are Ailevate AMI Agents?

Ailevate AMI (Artificial Machine Intelligence) Agents are specialized AI assistants designed to perform specific tasks or serve particular domains. These agents are powered by large language models (LLMs) but are configured with specific capabilities, knowledge bases, and behaviors to make them experts in their designated areas.

Key Concepts

  • Agent: An AI assistant with a specific purpose and set of capabilities
  • Capabilities: The specific skills and functions an agent can perform
  • Knowledge Base: Domain-specific information the agent has access to
  • Model Settings: Configuration of the underlying AI model
  • YAML Configuration: The structured format that defines an agent's properties

Agent Creation Process

Creating a new agent involves several steps:

Step 1: Choose a Template

Start with a pre-configured template or build from scratch. Templates provide a foundation with pre-selected capabilities and settings for specific use cases.

Step 2: Define Basic Information

Set your agent's name, description, and category. This information helps users understand what your agent does and how it can assist them.

Step 3: Configure Capabilities

Select the specialized skills and knowledge bases your agent will have. These determine what tasks your agent can perform and what information it can access.

Step 4: Adjust Model Settings

Configure the underlying AI model parameters such as model type, context size, and temperature. These settings affect how your agent processes information and generates responses.

Step 5: Preview and Export

Test your agent's behavior through simulation and export the YAML configuration. This configuration can be imported into Dify.ai for deployment.

Model Settings

Model settings determine how your agent processes information and generates responses:

Model Type

The type of AI model that powers your agent:

  • LLM (Large Language Model): For text generation and conversation
  • Text Embedding: For semantic search and retrieval
  • Rerank: For reordering search results by relevance

Base Model

The foundation model to use for your agent:

  • Claude 3 Opus: Highest capability model, best for complex tasks
  • Claude 3 Sonnet: Balanced performance and efficiency
  • Claude 3 Haiku: Fastest and most efficient model
  • GPT-4: Advanced capabilities with strong reasoning
  • GPT-3.5 Turbo: Efficient model for general-purpose tasks

Key Parameters

Parameter Description Typical Range
Context Size Maximum context length the model can process 1,000 - 100,000 tokens
Temperature Controls randomness in output (higher = more creative) 0.0 - 1.0
Top P Controls diversity of output (nucleus sampling) 0.0 - 1.0
Max Tokens Maximum length of generated responses 100 - 4,096 tokens

Advanced Features

  • Agent Thought: Enables reasoning capabilities
  • Vision: Enables image understanding
  • Tool Usage: Enables interaction with external tools and APIs

YAML Structure

The YAML configuration defines all aspects of your agent. Here's an example of the structure:

model: agent-identifier
label:
  en_US: Agent Display Name
model_type: llm
features:
- agent-thought
- tool-use
description:
  en_US: Detailed description of what the agent does
knowledge_bases:
- knowledge-base-1
- knowledge-base-2
model_properties:
  mode: chat
  context_size: 16000
parameter_rules:
- name: temperature
  use_template: temperature
  default: 0.7
- name: top_p
  use_template: top_p
  default: 0.9
- name: max_tokens
  use_template: max_tokens
  default: 1024
capabilities:
  capability_1:
    enabled: true
    description: Description of the capability
  capability_2:
    enabled: true
    description: Description of the capability
tools:
  tool_1:
    enabled: true
    description: Description of the tool
  tool_2:
    enabled: true
    description: Description of the tool
pricing:
  input: '0.00'
  output: '0.00'
  unit: '0.000001'
  currency: USD

Key Sections

  • model: Unique identifier for your agent
  • label: Display name in different languages
  • model_type: Type of AI model (llm, text-embedding, rerank)
  • features: List of enabled advanced features
  • description: Detailed description of what the agent does
  • knowledge_bases: List of knowledge sources the agent can access
  • model_properties: Core model configuration
  • parameter_rules: Model invocation parameters
  • capabilities: Specialized skills the agent has
  • tools: External tools the agent can use
  • pricing: Cost information for usage

Dify.ai Integration

The Ailevate AMI Agent Creator generates YAML configurations that are compatible with Dify.ai, allowing you to deploy your custom agents to a production environment.

Integration Steps

  1. Export the YAML configuration from the AMI Agent Creator
  2. Log in to your Dify.ai account
  3. Navigate to "Models" section
  4. Select "Add Custom Model"
  5. Upload the YAML configuration file
  6. Configure any additional settings
  7. Deploy your agent
Note: Make sure your Dify.ai environment has access to the required AI models and knowledge bases specified in your configuration.

Best Practices

Follow these recommendations to create effective specialized agents:

General Guidelines

  • Focus on a specific domain: Create agents with a clear, focused purpose rather than trying to make them do everything.
  • Select appropriate capabilities: Only enable capabilities that are relevant to your agent's purpose.
  • Use descriptive names: Give your agent a name that clearly communicates its purpose and function.
  • Write detailed descriptions: Help users understand exactly what your agent can do with clear descriptions.

Model Settings

  • Match the model to the task: Use more capable models (like Claude 3 Opus) for complex tasks, and lighter models for simpler tasks.
  • Adjust temperature appropriately: Use lower values (0.0-0.4) for factual tasks, higher values (0.6-0.9) for creative tasks.
  • Set appropriate context size: Ensure your context size is large enough to handle your use case without being excessive.

Testing

  • Use the simulation feature: Test your agent with realistic queries before deployment.
  • Try edge cases: Test how your agent handles unusual or unexpected inputs.
  • Iterate based on feedback: Refine your agent configuration based on testing results.
